# Архитектура MyMeteo

## Цель

Сделать понятное мобильное приложение для метеочувствительных пользователей. Первая версия работает без платного backend и без API-ключей.

## Слои

### UI

`MainActivity` отвечает за:

- главный экран;
- дневник;
- профиль;
- рекомендации;
- график индекса;
- блок персонального риска;
- дисклеймер;
- настройки уведомлений;
- навигацию;
- запуск обновления прогноза.

UI сделан программно на Java. Это упрощает запуск учебного проекта без AndroidX и Material Components.

### Data

Пакет `data`:

- `WeatherApiClient` - получает погоду из Open-Meteo;
- `SpaceWeatherApiClient` - получает Kp index из NOAA SWPC;
- `ForecastRepository` - объединяет источники данных и risk-engine;
- `ProfileStore` - хранит профиль;
- `DiaryStore` - хранит дневник;
- `MyMeteoDatabase` - локальная SQLite-база приложения;
- `HttpJsonClient` - простой HTTP-клиент на `HttpURLConnection`.

### Domain

Пакет `domain`:

- `RiskEngine` - расчет индекса риска;
- `FactorStatus` - состояние отдельного фактора риска;
- `WeatherSnapshot` - погода;
- `GeomagneticSnapshot` - геомагнитные данные;
- `UserProfile` - профиль пользователя;
- `DiaryEntry` - запись дневника;
- `RiskForecast` - итоговый прогноз;
- `PersonalRiskModel` - простая локальная модель персонализации по дневнику;
- `PersonalRiskPrediction` - результат персонального прогноза;
- `RiskLevel` - уровень риска.

### Notifications

`MyMeteoNotifier` создает канал уведомлений и показывает предупреждение, если риск не низкий.

## Risk-engine

Алгоритм первой версии прозрачный и объяснимый:

- перепад температуры 7-10 градусов повышает риск;
- скачок давления 4,5-7,5 мм рт. ст. повышает риск;
- Kp index 5 и выше повышает риск;
- влажность выше 80% или ниже 25% повышает риск;
- профиль пользователя добавляет персональный вес.
- каждый фактор получает тэг, по которому выбираются рекомендации и уведомления.

Результат:

- `LOW` - спокойный день;
- `MEDIUM` - стоит быть внимательнее;
- `HIGH` - высокий метеориск.

## ML-персонализация

`PersonalRiskModel` обучается на локальных записях дневника. Модель очищает записи, преобразует погоду в признаки, разделяет дни на плохие и обычные по оценке самочувствия, рассчитывает веса факторов и проверяет точность на отдельной части данных.

Если по текущим погодным условиям найден повышенный персональный риск, главный экран показывает блок `Ваш персональный риск выше среднего`. В блоке выводятся оценка модели, главный личный фактор, точность проверки и краткое объяснение.

Модель сделана простой и локальной, без внешних ML-библиотек. Это позволяет показать принцип персонализации в учебном MVP и не усложнять проект платными сервисами или серверной инфраструктурой.

## Ограничения MVP

- Город по умолчанию - Москва.
- Данные профиля и дневника хранятся локально в SQLite.
- Нет аккаунтов и синхронизации.
- Нет медицинской диагностики: рекомендации носят справочный характер.
