# Соответствие приложения требованиям документов

## 1. Исследование и аналитика

Закрыто в приложении:

- учитываются перепады атмосферного давления;
- учитываются перепады температуры за сутки;
- учитывается влажность;
- учитывается геомагнитная активность через Kp-index;
- профиль разделяет пользователей по группам: гипертония, гипотония, мигрени, суставные боли, чувствительность к магнитным бурям.

Источники данных:

- Open-Meteo API - температура, влажность, давление;
- NOAA SWPC - планетарный Kp-index.

## 2. Проектирование функционала

Закрыто в приложении:

- есть единый индекс риска 0-100;
- есть шкала уровня риска: зеленый, желтый, красный;
- есть метео-дневник;
- запись дневника автоматически связывается с погодой, давлением в мм рт. ст., влажностью, Kp-index и текущим риском;
- есть быстрые кнопки симптомов;
- есть профиль пользователя;
- есть настройки уведомлений по отдельным триггерам;
- есть компактный график динамики индекса на сегодня и завтра.

## 3. Разработка контента и рекомендаций

Закрыто в приложении:

- рекомендации формируются по тэгам факторов: `pressure`, `temperature`, `humidity`, `geomagnetic`, `profile`;
- советы короткие и написаны в спокойном тоне;
- есть юридический дисклеймер при первом запуске и в разделе рекомендаций;
- источники рекомендаций указаны в приложении: ВОЗ, Минздрав РФ, РЛС;
- советы не назначают лекарства и не заменяют врача.

## 4. Техническая реализация

Закрыто в приложении:

- реализован локальный аналог backend-логики через `ForecastRepository`;
- данные автоматически запрашиваются из бесплатных API;
- `RiskEngine` сопоставляет сырые данные с алгоритмом риска;
- UI выполнен в антистресс-стиле с основными цветами `#212842` и `#F0E7D5`;
- добавлены локальные уведомления;
- добавлен fallback на demo-данные, если сеть недоступна.

## 5. ML / алгоритм персонализации

Для студенческого MVP реализована подготовка к ML:

- дневник хранит самочувствие и погодные условия в момент записи;
- аналитический блок показывает, сколько записей уже накоплено;
- после 14-30 дней записей можно использовать эти данные для поиска персональных триггеров.

Полноценная ML-модель оставлена как следующий этап, потому что для корректного обучения нужен накопленный пользовательский датасет.

## 6. Ограничения MVP

- Нет удаленного backend-сервера и аккаунтов: это осознанное решение для бесплатного студенческого проекта.
- Нет платных API и сервисов.
- Геолокация задается через город и координаты в профиле, чтобы не усложнять MVP разрешениями и картами.
- Хранение данных локальное в SQLite, без отправки медицинской информации на сервер.
